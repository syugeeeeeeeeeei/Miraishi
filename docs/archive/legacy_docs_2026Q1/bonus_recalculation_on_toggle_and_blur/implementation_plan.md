# 実装計画

## 1. 再計算トリガーの責務整理

- `ScenarioCard` 側に保存・再計算処理を集約する。
- `ScenarioInputForm` は UI イベント（ボタン切替・blur）を親へ通知するだけにする。

## 2. ボーナス切替時の即時反映

- ボタンの `onClick` で親の `onBonusModeSwitch` を呼び、
  - ローカル編集状態更新
  - シナリオ保存
  - 再計算
  を即時実行する。

## 3. 切替先フォームの blur 反映

- 固定額入力（円）と基本給連動入力（ヶ月）の `onBlur` で親の `onBonusInputBlur` を呼ぶ。
- blur時に現在の編集状態を保存・再計算する。

## 4. 重複再計算の抑制

- 入力欄からモード切替ボタンへフォーカス移動した場合、blur起因の再計算はスキップし、ボタン切替側の再計算のみ実行する。

## 5. 検証

- 型チェックと既存計算テストを実行し、変更の副作用がないことを確認する。
