# 実装計画

1. `ScenarioCard` のレイアウトを固定最小幅からレスポンシブ構成へ変更する。
- 2 カラム固定の `minmax(720px, ...) + minmax(460px, ...)` を廃止する。
- 広い幅では 2 ペイン、狭い幅では上下分割に切り替え、破綻を防ぐ。

2. ペイン背景と境界線を調整する。
- 入力側・結果側の背景を薄いグレーにし、内部カードの白背景が浮き出る状態にする。
- ペイン間の境界線をブレークポイントに応じて縦線/横線で切り替える。

3. `ScenarioInputForm` のカードスタイルを共通化して強化する。
- 白背景 + 境界線 + 影のカードスタイルを共通定義する。
- 各入力セクションに同一スタイルを適用し、視認性を統一する。

4. 狭幅時に崩れやすい入力レイアウトを調整する。
- 3 カラム化のブレークポイントを後ろ倒しする。
- 横並び行の `HStack` に折り返しを許可する。

5. 検証
- `yarn typecheck`
- `yarn test`
- `yarn build`
