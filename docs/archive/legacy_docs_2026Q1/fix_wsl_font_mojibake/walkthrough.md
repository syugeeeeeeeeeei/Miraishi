# 修正内容の確認

## 原因

- ソース文字列のUTF-8不整合は確認されなかった。
- 文字化けの主因は、Chakra の既定フォント設定（`Heading` など）がシステムフォントを優先し、環境によって日本語表示品質がぶれること。
- さらに、`@fontsource` の全サブセット読み込みはフォント資産数が多く、環境差による読み込み揺らぎを起こしやすい。

## 実施修正

1. `theme` に `fonts.body` / `fonts.heading` を追加して、日本語対応フォントスタックを全体適用。
2. `Zen Maru Gothic` の読み込みを `japanese-400/700` に変更。
3. `M PLUS Rounded 1c` の読み込みを `japanese-700` に変更。

## 結果

- `Heading` を含む主要テキストがテーマ経由で統一され、環境依存フォントへのフォールバックが起きにくくなった。
- ビルド成果物のフォント資産は多数の分割ファイルから少数の日本語サブセットへ整理された。
- `typecheck` / `test` / `build` はすべて成功。
