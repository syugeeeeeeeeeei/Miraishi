# 修正内容の確認

## 変更概要

固定残業代を「手入力の固定額」ではなく「基本給連動の自動計算」に統一した。
あわせて、シナリオ入力の基準を想定初任給として理解しやすいUIに調整した。

## 主要変更

- 型定義更新
  - `fixedOvertime.amount` を削除。
  - `probation.fixedOvertime` を削除。
- バリデーション更新
  - Zodスキーマから上記項目を削除し、型との不整合を解消。
- 入力UI更新
  - 「基本給（月額）」を「想定初任給（月額）」へ変更。
  - 試用期間の固定残業代入力欄を削除。
  - 固定残業代カードを「みなし時間入力のみ」に変更。
  - 固定残業代は自動計算される旨の補助説明を追加。
- 初期シナリオ更新
  - 不要な固定額フィールドを初期値から削除。
- 計算フロー表示更新
  - 固定残業代の算出式（基本給連動）を表示。
  - 翌年反映ルールの成長係数表示を実際の定義に合わせて明確化。
- テスト更新
  - 固定残業代が年次成長で増加する回帰テストを追加。

## 検証結果

- `yarn test --run src/main/lib/calculator.spec.ts` : 成功（6 tests passed）
- `yarn typecheck` : 成功（node/webともにエラーなし）

## 補足

既存データに旧フィールドが残っていても、新しい入力UIでは編集対象外となる。
今後の新規データ・更新データは新仕様（基本給連動のみ）に沿って扱われる。
