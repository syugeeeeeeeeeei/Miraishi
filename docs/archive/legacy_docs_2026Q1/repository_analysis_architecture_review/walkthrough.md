# 修正内容の確認（分析ウォークスルー）

## 実施内容

1. リポジトリ全体を確認し、アプリが Electron + React + Jotai ベースのデスクトップ給与シミュレーターであることを確認。
2. `main / preload / renderer` を読み、主要フローを確認。
3. 型定義と税制スキーマを確認し、ドメインモデルの意図と実装差分を抽出。
4. `yarn typecheck`, `yarn test`, `yarn build` を実行し、ビルド・テストの成立を確認。

## 主な把握結果

- 主目的: 将来給与（額面/手取り）の可視化によるキャリア意思決定支援。
- 中核機能: シナリオ管理、詳細入力、税社保考慮の予測計算、テーブル表示、比較グラフ表示。
- 強み: 計算ロジックを Main に集約し、UI と分離。税制を JSON 外部化。

## 主要リスク（要修正候補）

- 変動残業 `enabled` を見ずに常に残業代を加算している。
- 手当の `percentage` が期限付きでも常時計上される。
- `months` 期限手当が年単位ロジックのため部分年を正確に表現できない。
- キャッシュが設定パターン分だけ増え続け、削除時にも無効化されない。
- 再計算の非同期呼び出しが重なった際に、古い結果で上書きされる可能性がある。
