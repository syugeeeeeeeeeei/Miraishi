# 実装計画

## 1. バックエンド対応

- 税金ルール取得/更新のIPCを追加する。
- 更新時に税金ルールをバリデーションし、保存後に計算キャッシュを無効化する。
- アプリ起動時はバンドルJSONを読み込みつつ、保存済み税金ルールがあれば優先して読み込む。

## 2. Preload API拡張

- Rendererから `getTaxSchema` / `updateTaxSchema` を呼べるように公開する。

## 3. UI導線追加

- コントロールパネル最下部に歯車付き円形メニューコンポーネントを追加する。
- 税金ルール、インフォ、クレジットのメニューを定義し、将来的な拡張を容易にする。

## 4. 税金ルールダイアログ

- ヘッダー/ボディー/フッター構成のモーダルを実装する。
- `closeOnOverlayClick=false`, `closeOnEsc=false` に設定して誤タップ閉じを防ぐ。
- フォームモードで税金スキーマ全体を編集可能にする。
- JSONエディットモードで全文編集/コピペを可能にし、構文・形式チェックを実施する。

## 5. 反映フロー

- ダイアログ内の変更はドラフト状態へ即時反映する。
- 「変更せず閉じる」はドラフト破棄。
- 「変更を確定」でIPC更新し、成功時に再計算を実行する。

## 6. 検証

- `yarn typecheck`
- `yarn test --run src/main/lib/calculator.spec.ts`
