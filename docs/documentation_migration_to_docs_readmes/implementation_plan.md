- `documents` を廃止し、`docs/readmes` を正本にする。
- 4文書（要件定義書/仕様書/用語定義書/取り扱い説明書）を再設計する。

## 2. 文書再設計

- 現実装に基づく内容を維持しつつ、画面配置やファイル名など可変要素への依存を削減する。
- 将来の機能追加に耐えるよう、コア契約と拡張ルールを明記する。

## 3. 物理移行

- `docs/readmes` に新規作成。
- `documents` は移行案内のみ残し、重複ソースを排除する。
