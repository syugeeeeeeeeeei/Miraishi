## このドキュメント群の設計方針

- 画面名・実装ファイル名・UI配置などの可変要素ではなく、プロダクトの本質的な責務と契約を中心に記述する。
- 新機能の追加や内部リファクタリングが行われても、既存の契約を壊さない限り本文の改訂を不要にする。
- 変更時の判断基準は「互換性ルール（仕様書）」に従う。
