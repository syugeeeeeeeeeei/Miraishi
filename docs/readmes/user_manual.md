# 取り扱い説明書（ユーザー向け）

## 1. この説明書について

本書は、Miraishi の最新UIに基づいた日常操作をまとめたものです。  
特に以下の更新を反映しています。

- 税金ルール設定を `YAMLエディター + 確認画面` に統一
- シナリオ入力に「初任給（固定残業代込み）」「固定残業時間」「年間休日数」を追加
- 比較レポート(PDF)のA4縦スライド形式出力を追加

## 2. できること

- 複数シナリオを作成し、年収額面・手取りを比較する
- 税制スキーマをYAML差し替えで更新する
- 税制スキーマの検証、差分確認、履歴復元を行う
- 複数シナリオを比較レポートPDFとして出力する

## 3. 画面の見方（概略）

- コントロールパネル: シナリオ作成、選択、検索、削除
- データビュー: シナリオ入力と結果確認
- グラフビュー: 複数シナリオ比較
- オプションメニュー（左下の歯車）: 税金ルール設定、比較レポートPDF出力

## 4. シナリオ入力の使い方

### 4.1 シナリオを作成する

1. コントロールパネルで「新しいシナリオ」を作成
2. シナリオ名を変更
3. 比較したいシナリオを複数選択

### 4.2 初期条件を入力する

主な入力項目:

- `初任給（固定残業代込み / 月額）`
- `固定残業時間 (h)`
- `年間休日数 (日)`
- `ボーナス`（固定額 / 基本給連動）
- `給与成長率`
- `勤務都道府県`
- `業種（雇用保険率）`
- `扶養・前年度収入`

入力補助として、フォーム内に次が自動表示されます。

- 所定労働時間（月）
- 算出基本給（月額）
- 算出固定残業代（月額）

### 4.3 保存と再計算

- 入力変更後は保存と再計算が自動実行されます。
- 都道府県・業種を変更した場合も再計算対象です。

## 5. 税金ルール（YAML）の更新

### 5.1 開き方

1. 左下の歯車アイコンを押す
2. `税金ルール` を選択
3. `税金ルール設定（YAML）` ダイアログを開く

### 5.2 編集から適用まで

1. `エディター` タブでYAMLを編集
2. `確認を更新` を押して検証
3. `確認画面` タブで以下を確認
   - 構文結果
   - 構造検証結果
   - 意味検証結果
   - 現行との差分サマリ
   - 適用対象バージョン情報
   - 項目定義・計算式（アコーディオン）
4. 問題がなければ `変更を適用`

重要:

- `変更を適用` は検証が `VALID` になるまで無効です。
- YAMLの未知キーや必須不足、税率帯の不整合、式エラーは適用できません。

### 5.3 履歴・復元

- 確認画面の `過去スキーマ履歴` で履歴を確認できます。
- 任意の履歴を `復元` すると、そのスキーマが有効化され再計算対象になります。

## 6. 比較レポート（PDF）出力

### 6.1 出力手順

1. 左下の歯車アイコンを押す
2. `比較レポート(PDF)` を選択
3. ダイアログで以下を設定
   - 出力対象シナリオ（2件以上）
   - 比較対象年（N年目まで）
   - 月平均残業時間
   - 出力セクション
4. `PDFを書き出す` を実行して保存先を選択

### 6.2 出力内容（現行）

A4縦のスライド形式で、主に以下を出力します。

- レポート表紙
- 各シナリオのサマリー
  - 初期入力条件
  - 主要KPI
  - 年次推移グラフ（軸ラベル付き）
- 各シナリオの年度ごとの給与詳細（2ページ）
  - 年収額面
  - 年収手取
  - 税金の内訳（色棒グラフ）
  - 月給額面
  - 月給手取
  - 前年比（額面/手取の増加額・増加率）
- 税制スキーマと計算前提

補足:

- 出力セクションのうち `税制スキーマ情報` は個別ON/OFFされます。
- それ以外のサマリー系セクションは現行版では同一グループとして扱われます。

## 7. よくある確認ポイント

- 比較レポート出力ボタンが押せない
  - シナリオが2件以上選択されているか
  - 比較対象年が 1〜50 か
  - 出力セクションが1つ以上ONか
- 税制スキーマが適用できない
  - 確認画面の `構文/構造/意味` エラーを順に解消
  - `VALID` 表示になっているか

## 8. 注意事項

- 本アプリは意思決定支援の試算ツールであり、将来結果を保証するものではありません。
- 制度改定時は税制スキーマを更新して再計算してください。
- 実運用では実際の給与明細・自治体通知・就業規則を優先してください。
