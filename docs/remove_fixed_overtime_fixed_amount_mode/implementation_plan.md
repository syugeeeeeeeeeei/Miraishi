# 実装計画

## 1. 仕様統一（型・バリデーション）

- `FixedOvertime` から `amount` を削除する。
- `Probation` から `fixedOvertime` を削除する。
- Zodスキーマを同内容へ更新し、入力モデルと検証モデルを一致させる。

## 2. 入力UIの整理

- 初任給入力ラベルを「想定初任給（月額）」へ変更する。
- 試用期間カードから固定残業代（月額）入力を削除する。
- 固定残業代カードは「制度ON/OFF + みなし時間」のみとし、月額は自動計算であることを明記する。

## 3. データ初期値の整合

- 新規作成シナリオの初期値から不要フィールドを除去する。

## 4. 表示説明の整合

- 計算詳細ダイアログに固定残業代の基本給連動式を表示し、固定額モード前提の誤解を防ぐ。

## 5. 回帰確認

- 固定残業代が給与成長に連動して増加するテストを追加する。
- `yarn test`（対象spec）と `yarn typecheck` を実行し、エラーがないことを確認する。
