# 実装計画

## 1. 互換性を保ったデータモデル更新

- 式の入力形式を `string | AST` の受理に変更する。
- 適用・保存時の既存データ互換を壊さない。

## 2. 安全な文字式コンパイル基盤

- `eval/new Function` を使わず、独自パーサで `AST` に変換する。
- 許可演算子/関数をホワイトリストで限定する。
- 未許可構文（プロパティアクセス、任意関数呼び出しなど）を拒否する。

## 3. セキュリティ制約

- 危険キー（`__proto__` 等）を式中・テーブル参照時に拒否。
- 式長・トークン数・ノード数・深さの上限を設定。
- 失敗時は例外として `apply` 拒否（fail closed）とする。

## 4. 正規化と可読性

- 既存AST式を読み込み時に文字式へ変換し、エディタ表示を改善する。
- デフォルト式定義を文字式へ移行する。

## 5. 検証

- 型検査: `yarn typecheck`
- テスト: `yarn test --run`
- 追加テスト: 未許可関数、危険キー、長大入力拒否、互換正規化
