# 実装計画

## 目的
現行の開発/ビルド/デプロイ手順を可視化し、シンプル化のための改善候補を優先度付きで整理する。

## 実施ステップ
1. 現状フロー把握
- `package.json` scripts から開発・ビルド・配布コマンドの実体を抽出
- `scripts/deploy-win.mjs` で Windows 配布分岐（wine 有無）を確認
- `electron-builder.yml` の配布設定を確認

2. ドキュメント整合性確認
- `README.md` の手順と scripts の一致確認
- 過去の配布関連ドキュメントとの差分確認

3. シンプル化候補の抽出
- スクリプト名/責務/既定値の整理
- 不要依存・未使用設定の候補抽出
- セキュリティと運用事故の観点を追加

4. 優先順位付け
- 即時対応（効果大・工数小）
- 次段対応（要件確定後）
- 現状維持（変更メリットが薄い）
